Java基础

1、JVM、JDK、JRE

JRE：Java运行环境，包括JVM，但是不能创建新程序。

JDK：Java开发工具包，功能最齐全。

JVM：Java虚拟机。

2、public、private、protected、default

public：一个源文件中只能有一个类是public。所修饰成员变量可在任何类中访问到。

protected：受到该类所在的包作为保护。所修饰成员变量只能在同一包中访问。

private：只能在当前类中被访问到。

3、final、finally、finalize

final：修饰变量时必须初始化，成为常量。

finally：常常与try catch搭配使用。附带的语句块一定会被执行，无论是否抛出异常。

finalize：每一个对象都有的方法，一个对象只会被其调用一次，但调用不代表回收。

4、static

只有new对象之后，数据存储空间才会被分配，方法才能由外界调用。使用static，成员变量或方法可以在没有创建对象的时候使用。

5、面向对象三大特性

封装：封装成抽象的类，对可信任的对象或类可以进行操作，对于不可信的进行隐藏。

继承：可以使用现有类的所有功能，并在此基础上还可以进行拓展。

多态：基于继承，指负累中定义的属性和方法被子类继承之后，可以具有不同的数据类型或表现不同的行为，使同一个属性在父类和子类中具有不同的含义。

实现多态的3个必要条件：继承、重写、向上转型

  • 继承：必须存在继承关系的子类和父类
  • 重写：子类对父类中的方法重新ing一，调用方法时使用子类中的方法
  • 向上转型：将子类的引用赋给父类对象，只有这样引用才既可以调用父类的方法也可以调用子类方法。、

6、重写和重载的区别

重写：运行时多态。发生在子类和父类之间，要求返回值和形参都不能改变。外壳不变，核心重写。两小(返回值类型、抛出异常)两同(方法名和参数列表)一大(访问权限)

重载：编译时多态。同一类里面，方法名相同但参数不同。返回类型也可不同。常用的就是构造器的重载。

7、ArrayList

底层是一个Object【】数组。底层数组默认初始化容量为10。

  1. 在jdk1.8中底层会先创建一个长度为0的数组。
  2. 当第一次添加元素使用add方法时，会初始化为一个长度为10的数组。

当容量使用完后会进行扩容

  1. 会自动创建容量更大的数组，并将原数组中所有元素进行拷贝，导致效率低下。
  2. 优化：使用构造方法提供一个初始化的容量。ArrayList(int capacity)/ensureCapacity(int capacity)。

三个构造方法：

  1. ArrayList()：创建初始容量为0的空列表
  2. ArrayList(int capacity)：指定一个初始容量的空列
  3. ArrayList(Collection<?extends E>c)：创建一个包含指定集合中所有元素的列表。

优点：

  1. 向末尾添加元素时效率较高
  2. 查询效率高

缺点：

  1. 扩容会造成效率低下。
  2. 数组无法存储大量数据。
  3. 向中间位置添加元素需要移动元素，效率低下。

8、LinkList

LinkedList底层是一个双向链表。

优点：增删效率高。

缺点：查询效率低。

LInkedLIst可以调用get(index)返回链表第index个元素，但是每次查找需要从头节点开始。

ListIterator接口
  1. LinkedList.add方法只能将数据添加到链表末尾。
  2. 可以使用ListIterator接口的add方法添加元素到链表中间位置。
    1. void add(E e)
    2. boolean hasNext()
    3. boolean hasPrevious()
    4. E next()
    5. int nextIndex()
    6. E previous()
    7. int previousIndex()
    8. void remove()
    9. void set(E e)
  3. remove方法：
    • 调用next后，删除的是迭代器左侧的元素。
    • 调用previous后，调用的是迭代器右侧的元素。

9、Vector

底层为数组，初始容量为10。扩容后是原始容量的2倍。

Vector是线程安全的，每个方法都使用了sychronized同步锁，效率低下。

10、泛型

好处：

  1. 减少强制类型转换次数
  2. 调用方法时更安全
  3. 泛型只在编译时起作用
  4. 在使用时没有指定泛型类型时默认使用Objext类型。

11、Set

HashSet

无序（没有下标），不可重复

TreeSet

无序（没有下标），不可重复，可以排序。

HashSet作为HashMap的key部分，TreeSet作为TreeMap的key部分。

12、Map

  1. Map和Collection没有继承关系
  2. Map以键值对的方式存储，key和value存储的都是对象的内存地址（引用）

常见方法：

  • Set keySet()
  • Collection values()
  • V put(key, value)
  • V get(Object key)
  • V remove (Object key)
  • boolean isEmpty()

遍历方法：

  • 第一类：先获取map的keySet，然后取出key对应的value。
    • 通过foreach遍历ma p.keySet()，取出对应的value。
    • 通过迭代器迭代map.keySet(),取出对应的value。
  • 第二类：调用map.entrySet()，获取entrySet，直接从中获取key和value
    • 调用map.entrySet(),然后使用foreach遍历
    • 使用迭代器遍历。
HashMap
  1. 底层由数组实现
  2. 数组中每个元素是一个单向链表
    • 链表每个节点时Node<k,v>类型
  3. 同一个单链表中所有Node的hash值不一定一样，但对应的数组下标一定一样
    • 数组下标是通过哈希函数计算出的
  4. HashMap是数组和单链表的结合体
    • 数组查询效率高但增删效率低
    • 单链表随机增删效率高，查询效率低
  5. 特点
    • 无序、不可重复(通过equals方法保证的)

源码解读：

  • 初试默认容量为16
    • 必须是2的2次幂，官方推荐
    • 需要达到散列均匀，提高几个的存取效率
  • 默认加载因子0.75
    • 数组达到容量的3/4开始扩容
  • 单链表元素超过8个转变为红黑树；红黑树节点小于6转变为单链表

put方法原理：

  • 先将key，value封装到Node对象中
  • 底层调用key的hashCode()计算hash值
  • 通过哈希算法，将hash值转换为数组下标
    • 如果下标没有元素，直接将Node添加进来
    • 如果下标有单链表，将当前Node中的key与链表上每一个节点中的key进行equals比较
      • 如果所有equals都返回false，则将Node添加至链表尾部
      • 如果其中一个返回了true，链表中对应节点的value将会被新Node中的value进行覆盖

注：

  1. HashMap允许key和value为null，但只能存在一个。
  2. HashTable中，key和value不允许为null。

get方法原理：

  • 先调用key的hashCode()得到hash值
  • 通过哈希算法将hash值转换为数组下标
  • 通过数组下标定位位置
    • 如果位置上没有元素，直接返回null
    • 位置上有单链表，此时会将当前Node中的key与链表上每一个节点中的key进行equals比较
      • 所有equals返回为false，get方法返回null
      • 有一个equals为true，get方法返回这个节点的value。

同时重写hashCode()和equals()方法。

  • 重写hashCode时要注意需要达到散列分布均匀
TreeMap

TreeSet/TreeMap是自平衡二叉树，迭代器采用的是中序遍历。

特点：无序，不可重复，但可以排序。

排序规则：

  • TreeSet/TreeMap中key可以自动对String或8大数据类型的包装类型进行排序
  • 无法对自定义类型进行排序，原因是没有实现java.lang.Comparable接口，解决自定义类型的排序有两种方式。
    • 实现java.lang.Comparable接口，重写compareTo方法。
    • 选择TreeSet/TreeMap带比较器参数的构造器，重写比较器中的compare方法。

13、String

  • string类型不可变，使用final修饰
  • 使用双引号的字符串都是直接存储在方法去的字符串常量池中

StringBuilder：底层数组没有用dinal修饰，可变；线程不安全，效率高

StringBuffer：也没有用final修饰，可变；线程安全，效率低。

Java集合

三个层次：最上层的接口、中间的抽象类和最后的实现类。

接口：Collection、List、Set、Map，以及辅助性质的接口Iterator、LinkIterator、Comparator

抽象类：实现大多数接口的方法，继承类只需要根据自身特性重写部分方法或者实现接口方法

算法：ArrayList、LinkedList、HashMap、HashSet等。

Java集合的基本接口是Collection和Map接口，所有实现类都是这两个接口派生下来的。Collection一次存一个元素是单列集合。Map一次存一对元素，是双列集合。

Collection接口

自身接口：

  • 增
    • add(E)
    • addAll(Collection)
  • 删
    • remove(Object)
    • removeAll(Collection)
    • removeIf(Predicate)
    • clear()
    • retainAll(Collection)
  • 获取
    • size()
  • 判断
    • isEmpty()
    • contains(Object)
    • containsAll(Collection)
    • Equals()
  • 方法
    • hashCode()
    • toArray()
  • 流
    • stream()
    • paralleIStream()

继承Iterable接口：

  • Iterator接口
    • hasNext()
    • next()
    • remove()
    • forEachRemaining()
  • forEach
  • spliterator

上面的方法在实现类中都必须提供，为了简化操作Java类提供了一个AbstractCollection。其又派生了AbstractList、AbstractSet等。

Java中常用的for each与发烫可以与任何实现了Iterable接口的对象一起工作。Iterable包含了Iterator对象，可以调用iterator()方法，返回一个Iterator对象，这就是一个常用的迭代器。

Iterator()功能简单，只能单向移动。

  1. 调用iterator()方法返回一个Iterator对象。
  2. 第一次调用next()返回序列的第一个元素。此后每调用一个next会返回下一个元素，在调用之前最好调用hasNext方法。
  3. remove删除上一次next返回的对象，只能在next之后使用。

List接口

List全部继承自AbstractList，是一个有序集合，允许重复元素。

List常用方法：

  • 增：add、addAll
  • 删：remove、removeAll
  • 改：set
  • 获取：get、subList、listIterator、indexOf、size
  • 判断：isEmpty、contains

其又特殊的迭代器接口ListIterator。是双向的，增加了previous()和hasPrevious()用来反向遍历。只能List使用，其他接口无法使用。

ListIterator中有：add、set、hasPrevious、previous、nextIndex、previousIndex。

Set接口

Set全部继承自AbstractSet，是无序集合，不允许重复。实现不允许重复就需要重写hashCode和equals方法。

Set常用方法：

  • 增：add、addAll
  • 删：remove、removeAll、clear、retainAll
  • 获取：size、toArray
  • 判断：isEmpty、contains、containsAll
  • 区别List：hashcode、equals

Queue接口

Queue常用方法：

  • 添加：add、offer
  • 返回头：element、peek
  • 删除：remove、poll

Deque接口继承自Queue，在Deque基础上细化，可以选择从头到尾操作。

Deque常用方法：

  • 添加：addFirst、addlast、offerFirst、offerLast
  • 删除：removeFirst、removeLast、pollFirst、pollLast、removeFirstOccurrence、removeLastOccurrence
  • 返回元素：getFirst、getLast、peekFirst、peekLast

Map接口

Map是顶层接口，sortedMap接口提供排序选项。所有实现类都派生于AbstractMap。其中有实现了排序、查找、替换、复制、数组工具类Arrays等很多功能的方法。

接口

接口概念：

  1. 接口中所有方法自动为public。
  2. 可以定义常量
  3. 没有实例字段

接口属性：

  1. 可以使用instanceof检查一个对象是否实现了某个接口
  2. 接口中的字段总是 public static final
  3. 多继承可以通过接口实现

接口和抽象类的异同

  • 同：1. 都允许进行方法的定义，不需要具体实现。2. 都允许被继承
  • 异：1. 抽象级别不同。接口中只允许方法的定义不允许方法的实现，抽象类中可以进行方法的实现。2. 使用的关键词不同。3. 接口中的变量都是公共的静态常量，抽象类中的变量可以是普通变量。

类加载机制

类的数据从class文件加载到内存，并对数据进行校验、转换解析和初始化，最终形成Java类型。

类的生命周期：

  • 加载：
    • 通过类全限定名获取定义此类的二进制字节流
    • 将字节流的静态存储结构转换为方法区运行时的数据结构
    • 在内存中生成一个代表类的java.lang.class对象，作为类各种数据的访问入口
  • 验证：
    • 文件格式验证
    • 元数据验证
    • 字节码验证
    • 符号引用验证
  • 准备：为类中静态变量分配内存并设置类变量初始值
  • 解析：Java虚拟机将常量池内的符号引用替换为直接引用的过程
  • 初始化：执行类构造器方法的过程
  • 使用
  • 卸载

双亲委派模型：一个类加载器收到类加载的请求，首先不会自己去尝试加载类，而是把请求委派给父类加载器去完成，每一层的类加载器都是如此，最终都传送到最顶层的启动类加载器。只有当父加载器反馈无法完成请求时，子加载器才会自己处理。

分为4层：

  1. 启动类加载器 Bootstrap class loader
  2. 扩展类加载器 Extension class loader
  3. 应用类加载器 Application class loader
  4. 自定义加载器

Java并发

Synchronized

作用：通过使用内置锁实现对共享变量的同步操作，结局共享变量操作的原子性，保证其他线程对共享变量的可见性、有序性，从而确保并发情况下的线程安全。

用法：

  1. 普通同步方法
  2. 静态同步方法
  3. 同步代码块

内置锁：Java中每个对象都有一把锁，防止在对象头中，用于记录当前对象被哪个线程池游。一个对象在堆中是由对象头、实例数据、填充构成。

内置锁的实现：使用monitorEntry和monitorExit两个字节码指令，实质是操作系统中的mutex lock实现。

无锁：不锁资源，多个线程中有一个能修改资源成功，其他线程会重试

偏向锁：一段同步代码、共享资源一直被一个线程访问，那么该线程自动获取锁，减少获取锁的代价

轻量级锁：当锁是偏向锁，被另外的线程访问就会升级为轻量级锁，其他线程会通过自旋的形式尝试获取锁，不会阻塞。

重量级锁：互斥锁，设计CPU的介入。

JVN

运行时数据区包含：

  • 共享
    • 方法区
    • 堆
  • 私有
    • 本地方法栈
    • 程序计数器
    • 虚拟机栈

私有：

  • 程序计数器：记录正在执行的虚拟机字节码指令的地址。字节码解释器工作时通过改变计数器的值获取下一条待执行指令的地址。多线程下每个线程都需要有一个独立的计数器，方便在切换时恢复到正确的执行位置。

  • 虚拟机栈：由很多个栈帧组合，每个方法执行时，虚拟机会同步创建一个栈，方法从调用到执行完毕对应的栈帧从入栈到出栈的过程。

  • 局部变量表：存放编译期间可知的各种虚拟机基本数据类型、对象引用等。

  • 本地方法栈：虚拟机使用到的本地方法native方法服务

公有

  • 堆：虚拟机启动时创建，是垃圾收集器管理的内存区域，这里是GC的发生地
  • 方法区：存储已被虚拟机加载的类型信息、常量、静态变量、编译器编译后的代码缓存等数据
  • 运行时常量池：包括类的描述信息和常量池表。运行时常量池具有动态性，运行期间可以放入新的常量

对象的创建：

  1. 类加载检查
  2. 分配内存
  3. 初始化内存空间
  4. 进行对象头的设置
  5. 执行构造函数

对象的结构

  1. 对象头：
    • 对象自身的运行时数据，如：Mark word、hash code、GC年龄
    • 类型指针：对象指向元数据的指针，通过这个确定对象是哪个类的实例
  2. 实例数据
  3. padding

对象访问定位

Java通过栈上的reference数据来操作堆上的具体对象

  • 句柄方式：堆中分配一块内存作为句柄池，reference储存句柄池的地址，句柄中包含了指向对象实例数据的指针和对象类型数据的指针。
    • 稳定，对象被移动时，只会改变句柄中的实例数据指针，不需要修改reference本身
  • 直接指针：reference存放的是对象地址
    • 访问速度快，减少一次指针定位的开销
